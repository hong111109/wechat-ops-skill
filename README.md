> 公众号不是流量容器，是信任资产。

面向个人 IP / 独立创作者的微信公众号运营全栈指南，覆盖从定位到转化的完整链路。

---

## 核心能力

| 模块 | 解决什么问题 |
|------|-------------|
| **定位与战略** | 账号方向、读者画像、差异化定位 |
| **选题与栏目** | 内容系统、栏目矩阵、发布节奏 |
| **写作与包装** | 标题、开头、结构、CTA 优化 |
| **分发与增长** | 转发机制、增长实验、流量获取 |
| **私域与转化** | 承接链路、咨询转化、变现设计 |
| **数据诊断** | 复盘框架、瓶颈定位、数据解读 |
| **轻协作** | 小团队配合、SOP、周会机制 |

---

## 四种工作模式

根据你的问题类型，自动匹配最适合的协助方式：

| 模式 | 适合场景 | 交付形式 |
|------|----------|----------|
| **顾问模式** | "我该怎么做" | 判断、优先级、关键取舍 |
| **操盘模式** | "直接帮我出方案" | 可执行内容、栏目矩阵、发布节奏 |
| **教练模式** | "帮我改这篇文章" | 逐项点评 + 改写示例 |
| **诊断模式** | "为什么没效果" | 瓶颈定位 + 验证动作 |

---

## 快速开始

**触发关键词**：公众号运营、公众号写作、选题、标题、栏目、私域承接、转化复盘

**示例对话**：

```
用户：我想做一个公众号，但不知道该写给谁看。
→ 进入顾问模式，提供定位策略

用户：直接帮我出一周选题和栏目安排。
→ 进入操盘模式，输出栏目矩阵 + 选题池

用户：这是我的标题和开头，帮我改得更容易被点开。
→ 进入教练模式，逐项点评并改写

用户：阅读量还行，但几乎没人加我。
→ 进入诊断模式，定位转化链路问题
```

## 核心原则

- 先看用户价值，再看数据表现
- 一篇内容只承担一个主目标
- 选题问题优先级高于文笔问题
- 增长动作不能破坏长期人设一致性
- 内容、关系、产品必须联动思考

---

## 反模式（避免这些）

- ❌ 把公众号运营做成标题党技巧库
- ❌ 把所有问题都归因到写作
- ❌ 只讲增长，不讲信任和转化
- ❌ 把个人 IP 和企业号混为一谈
- ❌ 把私域转化写成硬广动作

---

## 适用人群

- **个人 IP / 独立创作者** — 想系统运营公众号，建立长期信任资产
- **小团队内容负责人** — 需要栏目规划、协作 SOP 和诊断框架
- **内容创业者** — 关注从内容到转化的完整链路

**不适用**：大型团队矩阵打法、纯流量玩法、微信规则百科查询

---

## 使用建议

1. **描述你的现状**：行业、读者类型、目前数据、遇到的具体问题
2. **明确你的目标**：想解决什么问题（定位/选题/写作/增长/转化/诊断）
3. **选择模式**：想被指导（顾问）、直接要方案（操盘）、改具体稿件（教练）、找问题根源（诊断）
