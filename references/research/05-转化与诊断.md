# 05 转化与诊断

## 核心判断
- 公众号运营本质是内容—关系—产品三者联动。
- 转化不是在文章末尾突然发力，而是从内容承诺开始铺路。
- 数据差时，先判断是选题问题、入口问题、体验问题、分发问题，还是承接问题。

## 常见转化链路
### 链路 A：公众号 → 私聊咨询
文章提供判断与案例 → CTA 引导回复关键词 / 添加方式 → 私聊诊断 → 预约咨询。

### 链路 B：公众号 → 社群 / Newsletter / 清单
文章提供方法或观点 → CTA 引导领取延伸资料 → 进入长期培育池。

### 链路 C：公众号 → 产品 / 服务页
文章先教育问题与方案边界 → CTA 指向产品说明页或服务页。

## 诊断树
1. 阅读量低：先看选题和标题，不先怪正文。
2. 打开还可以但读完差：先看开头和结构。
3. 读完反馈不错但没动作：先看 CTA 和承接链路。
4. 有流量但没成交：先看 offer 匹配度与私域承接，而不是继续堆流量。

## 复盘模板
- 这篇文章的主目标是什么？
- 最终有没有完成这个主目标？
- 最大阻力出现在进入、读完还是下一步动作？
- 下次只改一个变量，改什么？

## 诊断模式默认输出
- 问题更可能出在哪一层
- 一次只改一个变量的下一轮动作
- 需要观察的核心信号
- 不建议现在改的东西

## 常见误判
- 误判 1：把所有结果问题都归因到流量不够
- 误判 2：没有承接设计，却期待文章直接成交
- 误判 3：在信任尚未建立时过早推转化
